## 集里怎么说它

- **[[2026-01-18-lennys-the-non-technical-pms-guide-to-building|《2026-01-18-lennys-the-non-technical-pms-guide-to-building》]]**(00:19起):本集把它说成：嘉宾在 Cursor 中运行的智能体（充当其「CTO」或开发主管），负责探索代码库、生成计划、执行写代码并自我审查。
- **[[2026-01-29-lennys-marc-andreessen-the-real-ai-boom|《2026-01-29-lennys-marc-andreessen-the-real-ai-boom》]]**(75:01起):本集将其描述为最近节日期间令编码圈震撼的杀手级应用，相当于一个驾驭大模型的智能体外壳（agent harness）；但它一周半就被开发出来，护城河被质疑。
- **[[2026-05-03-lennys-why-cultivating-agency-matters-more|《2026-05-03-lennys-why-cultivating-agency-matters-more》]]**(09:12起):本集提到，这是鼓励设计师和产品经理使用的终端编码工具之一，帮助他们直接通过代码表达原型。
- **[[2026-05-24-lennys-the-ai-paradox-dan-shipper|《2026-05-24-lennys-the-ai-paradox-dan-shipper》]]**(00:00起):本集说它是 Anthropic 推出的、能在终端里运行代码的智能体，能通过访问终端获得电脑数据的超级权限，擅长处理非工程类工作。
- **[[2026-06-28-lennys-openai-codex-lead-on-the-new-shape|《2026-06-28-lennys-openai-codex-lead-on-the-new-shape》]]**(16:51起):嘉宾将其作为对比对象，指出它待在本地终端、不假装是 AGI 而是不断向人类提问，这种更谦卑的形态因为契合了当时模型的能力边界而在市场上大获成功。
- **[[2026-singju-openclaw-80apps|《OpenClaw创始人：为何80%的应用将消失》]]**(02:55起):本集讨论的 Anthropic 推出的同类命令行编码智能体。嘉宾持反向观点，认为全世界都在用但他“不觉得可以用 Claude Code 构建任何东西”，并提及初版 OpenClaw 曾在连接 WhatsApp 和 Claude Code 的依赖间加了点“胶水”。
