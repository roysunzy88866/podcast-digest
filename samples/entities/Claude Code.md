## 集里怎么说它

- **[[2026-03-01-lennys-the-design-process-is-dead|《2026-03-01-lennys-the-design-process-is-dead》]]**(00:52起):本集把它说成：用于搭配 VS Code 调整前端细节的工具；能极快地编写代码并提交修复，甚至开始帮助用户产生想法、决定要构建什么功能。
- **[[2026-03-29-lennys-how-openclaw-changed-my-life-claire-vo|《2026-03-29-lennys-how-openclaw-changed-my-life-claire-vo》]]**(41:34起):嘉宾把它当成 OpenClaw 的「外科医生」和「上帝模式管理员」。因为 OpenClaw 本质是配置代码，遇到问题只需让同一机器上的 Claude Code 读文档、修配置，甚至用它来拆分、移植老智能体的记忆。
- **[[2026-04-05-lennys-anthropics-1b-to-19b-growth-run|《2026-04-05-lennys-anthropics-1b-to-19b-growth-run》]]**(09:03起):本集把它说成是Anthropic内部的一款智能体编程工具，它给工程师带来了指数级的生产力杠杆，同时也挤压了PM和设计师的精力，促使团队必须让工程师充当「迷你PM」的新工作流。
- **[[2026-05-24-lennys-the-ai-paradox-dan-shipper|《2026-05-24-lennys-the-ai-paradox-dan-shipper》]]**(00:00起):本集说是 Anthropic 推出的命令行编程智能体，嘉宾一年前精准预测了它在非技术工作（如整理文件、写文章）上的巨大潜力。
- **[[2026-06-21-lennys-building-the-most-ai-pilled-engineering|《2026-06-21-lennys-building-the-most-ai-pilled-engineering》]]**(01:07起):本集核心讨论的 AI 辅助编码产品，它让编码不再是瓶颈。不仅工程师，连 PM 和设计师都在用它提交代码，并通过挂载在所有代码库上的远程会话帮助管理者掌握团队工作进度。
- **[[2026-singju-openclaw-80apps|《OpenClaw创始人：为何80%的应用将消失》]]**(02:55起):本集讨论的 Anthropic 推出的同类命令行编码智能体。嘉宾持反向观点，认为全世界都在用但他“不觉得可以用 Claude Code 构建任何东西”，并提及初版 OpenClaw 曾在连接 WhatsApp 和 Claude Code 的依赖间加了点“胶水”。
