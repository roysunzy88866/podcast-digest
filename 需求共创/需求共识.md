## 边界确认 Boundary Confirmation
> 必填 6 全 + 总数 ≥7 才许画。

- **WHO** (谁用): 用户本人 + 身边朋友/同事的小圈子；非技术运营者视角（用户自述看不懂代码，产品要能自己维护订阅源）。【🔒】 ← 必填
- **AUDIENCE** (给谁看): 同 WHO——自己消费为主，产出要拿得出手转给身边人。【🔒】 ← 必填
- **PAIN** (真痛点): X / YouTube 上的优质英文长视频（技术、AI coding、商业模式类访谈）太长看不完；市面「声音克隆 + 整集翻译成中文」的产品，一小时音频转出来质量不高、听感差。现状凑合法：硬啃原片、攒着不看，或忍受低质量整集翻译。【🔒】 ← 必填
- **FORM** (长啥样): ① 网页小站（文字 + 音频都在上面，自动更新，发链接即可分享，可挂 hearagain.space 子域）② 播客 RSS（音频进小宇宙 / Apple Podcasts 等播客 App 通勤听）。不做即时推送（现阶段）。【🔒】 ← 必填
- **INPUT** (怎么进数据): 只自动订阅，两类源：① **播客 RSS（首选）**——目标节目基本都是播客，公开 RSS + 音频直链下载，全白色通道（播客 App 的标准工作方式），发现+获取一次解决【📌 第 4 轮发现，架构默认】；② **YouTube 频道**（官方 RSS/WebSub 发现 + yt-dlp 获取）只兜底 YouTube 独占内容（如 AI Engineer 大会演讲）。不做手动丢链接。**X 自动订阅推迟到二期**（重大判断，第 3 轮拍板）。**订阅源清单 v1 已拍板**（12 个，见「订阅源清单 v1」节）🔒。【🔒】 ← 必填
- **OUTPUT** (输出啥): 每期长视频 → 一份中文精华：文字打底（5-10 分钟能读完），音频从确认过的文字生成（短音频，非整集）。具体分层形态（TLDR / 精华稿 / 原文对照）沿 A+B+C 融合方向细化中 💡。【🔒】 ← 必填
- **ENGINE** (核心机制): AI 流水线：抓取 → 转写 → 浓缩 → 产出（文字+音频）。浓缩机制方向已拍板 = A+B+C 三路线融合（原声精选+中文串讲 / 编辑部式提炼稿 / 金字塔分层+可溯源），具体怎么融合待调研后收敛 ❓。【🔒方向 / ❓细节】
- **CONTEXT** (用在哪): **手机 + 桌面并重** 🔒（第 12 轮拍板）——通勤听音频/手机读 与 桌面细读 都是主场景；每个页面画两套线框（手机竖 32×60 + 桌面横 70-90×28）。频率取决于订阅源更新量（约 15-20 期/周，过滤后更少）。【🔒】
- **CONSTRAINTS** (约束): ⓪ **用户自有资源**：GLM API（智谱 key，glm-ask/glm-check 已配）、Azure/Cloudflare 账号——成本方案围绕它们设计【🔒】。① **架构 = 全云端，无 Mac mini**【🔒 第 27 轮改】：因输入改「播客 RSS 首选」（公开直链、无需住宅 IP），Mac mini 最大存在理由（住宅 IP + 本地免费转写）消失。整条流水线放 Cloudflare（Workers 定时 + 调云 API + R2 + Pages），无常开机器。② **转写环节相应改**：优先用官方文字稿（头部播客多自带，质量高于 Whisper、免费）；无官方稿的调云端 ASR（Groq ~¥0.3/小时无分离 / AssemblyAI ~¥1.2/小时含分离，送 $50 起步额度）。③ **成本**：从 ≈¥0 变 **≈¥0 到几十块/月**（视多少源自带文字稿，大概率贴近零，前几月免费额度可全免）。④ **一个坑**：YouTube 独占内容（yt-dlp 需住宅 IP+运行时）Cloudflare 跑不了→兜底/少数，第一版先靠播客源，这类手动或跳过。⑤ X 侧二期【🔒 第 3 轮】。⑥ 版权：ToS 灰色自担【🔒】。~~私有分发被追责概率极低~~ ~~音频文件公开输出→私密性靠 Cloudflare Access~~ **【2026-07-19 用户明文改 · drift #17 · ADR 0010】站点完全公开访问(可被搜索引擎收录）——「小圈子私有」是受众定位(自用+朋友)不是访问加密，原「靠 Access 登录门」是过度设计已删；公开使版权敞口从私有小圈子扩大到公开二次分发，用户明文接受风险自担。**
- **NOT** (不做啥 + 为什么):
  - 不做 D「中文双人播客重演」— 离原话最远、失真风险最高，正是用户担心的「四不像」重灾区。【🔒】
  - 不做手动丢链接投递 — 用户明确选了只自动订阅（将来可作扩展，非现阶段）。【🔒】
  - 不做整集完整翻译/声音克隆 — 市面已有且质量不达预期，本产品定位就是「精华版」而非「翻译版」。【🔒】
  - 不做即时推送（微信/TG/邮件）— 用户选了网页小站 + 播客 RSS 两种送达。【🔒】

**覆盖度**: 8/10  (必填 6/6)

## 产品方向备选
> 脑洞产物(如触发)。

- **D 中文双人播客重演**（NotebookLM 式）：听感最自然，但失真风险最高 → 用户未选，已进非目标。【🔒】
- **备忘**：A+B+C 用户全选，三者可叠成一个金字塔：C 分层可溯源当骨架，B 编辑部提炼稿当中层，A 原声精选+中文串讲当音频形态。具体融合方案待调研结果回来后与用户收敛。【💡】
- **「中文旁白+英文原声金句混排」音频形态**：调研发现全自动化产品无人做、原声零失真，曾推荐；用户第 3 轮拍板选纯中文 TTS（听感统一），混排存此备选，将来想试成本很低（每期几块钱）。【🔒 已决，备选留痕】

## 行业调研
> 守调研铁律:每条带链接 + 日期。三份原始报告（每条结论带链接+日期+证据分级）存档于本文件夹：
> `调研-同类产品摸底.md` · `调研-内容获取可行性.md` · `调研-技术管线与防失真.md`（均为 2026-07-15 调研）

**跨报告汇总结论**：

1. **差异化成立**【已知事实，基于三路检索】：「自动订阅 → 浓缩成 5-10 分钟中文精华（文字+短音频）→ 网页+播客 RSS」的完整组合未搜到现成产品。市面「浓缩」与「溯源」是分裂的：做浓缩的溯源弱（NotebookLM 音频无引用），溯源强的不浓缩（沉浸式翻译/Snipd/播客翻译计划）。**「摘要级溯源」（每条中文精华挂时间戳回链+关键论断附原文引述）市面没人做**。
2. **两个已验证的失败模式要避开**：全长翻译不起量（小宇宙「播客翻译计划」仅 2458 订阅）；纯 AI 选段废片率高（Opus Clip 用户实测约 40% 扔掉）→ 浓缩必须有质量闸门，不能一个 prompt 出稿。
3. **获取层结论**：YouTube 发现层完全白色（官方 RSS/WebSub，免费合规）；内容层灰色但个人低频+住宅 IP 基本无痛（跑云上接近必然被封）。**X 侧比 YouTube 难一个数量级**（免费 API 已死，按量付费 $5-30/月；灰色通道随时报废）→ 引出「X 侧策略」重大判断。
4. **技术层结论**：转写要自己做（YouTube 字幕无说话人标签，金句会张冠李戴）；浓缩用长上下文一口吞整读+强制时间戳引用（学术证据：map-reduce 层级合并做不出忠实摘要）；防失真两级闸门=字符串匹配（引语逐字+时间戳包含，机器卡死）→ LLM 裁判（忠实度审）。
5. **成本结论**【已知事实，官方定价页口径】：每期 2 小时访谈全管线（转写+浓缩+TTS）主流配置约 **¥4-5/期**，全用贵档也 <¥20/期。**第 5-6 轮结合用户自有资源后重构**：Mac mini 本地转写分离（免费含 diarization）+ DeepSeek 浓缩 + GLM 裁判 + **Azure Speech F0 免费层 TTS**（50 万字符/月永久免费、官方神经音色，全量 60-80 期/月只用约 24 万字符）+ Cloudflare 免费档分发 → **实际月成本 ¥5-20**（[Azure 定价页](https://azure.microsoft.com/en-us/pricing/details/speech/)、[2026 汇总](https://texttolab.com/blog/azure-text-to-speech-pricing)，2026-07-15 查）。另查：Cloudflare Workers AI 免费档 10k Neurons/天≈3.5h 转写/天，但**无说话人分离、中文 TTS 仅 MeloTTS**（[定价页](https://developers.cloudflare.com/workers-ai/platform/pricing/)、[模型目录](https://developers.cloudflare.com/workers-ai/models/)）→ 故转写留在 Mac mini 本地。
6. **分发层注意**：私有播客 RSS 用 Cloudflare R2+静态 feed 成本≈0，Apple Podcasts/Overcast 支持按 URL 订阅；**小宇宙不适合私有 feed**（贴入的 feed 会被平台抓取入库，私有性存疑【后半句未验证】）。
7. **竞争威胁**：Podwise 和 NotebookLM 补齐各自短板的成本都不高 → 速度和「技术/AI/商业访谈」垂类选题品味比功能本身重要。

## 核心定位          ← 叙事(可读,truth:why/what/value)
**v2（第 13 轮定位升级 🔒）**：不是「读完即走的内容站」，而是**「持续积累、互相串联的行业精英知识库」**——每期精华是知识网络的一个节点：概念能链到别期讲同一件事的内容，人物页聚合一个人在所有内容里说过的话。用户原话：**「在一个非常大的行业中，最精英人士的知识库演讲精华，并且它们之间可以相互串联起来」**「我不想让所有的知识成为孤岛」。
（v1「私人内容站」定位仍是子集：自动订阅→中文精华→网页+播客，不变；升级的是内容之间的组织方式。）

## 价值主张 Value Prop   ← 叙事(可读,核心定位的展开)
- **现状怎么凑合**: 硬啃英文原片 / 攒着不看 / 忍受一小时低质量整集翻译。【🔒】
- **用完变成什么样**: 订阅源出了新视频，第二天通勤路上 10 分钟听完/读完精华，金句还是原话没走样。【💡】
- **凭什么是你**: 市面产品做「整集翻译」或「无溯源摘要」，我们做「浓缩 + 摘要级溯源」——每句精华挂时间戳可回原片、金句中英对照。调研证实这个组合市面没人做（见行业调研 §1）。【📌 调研支持】
> ⚠️ **主张校准（第 7 轮，依据 调研-裁判可靠性.md）**：**「不失真」是夸大，不能这么说。** 证据：LLM 裁判 TNR<25%（抓不出失真）、零样本 prompt 天花板 63.4% F1、忠实度检测 SOTA 榜首也仅 77.4%、机器闸门保的是转写稿不是原话。
>
> **最终主张（第 7 轮用户拍板的两个选择共同决定）**：
> ### 「精华可能有偏差，但原文和全译完整保留，每句可点回原处，随时能查」【❓ 待用户确认表述】
>
> **防失真哲学 = 事后可查 ＞ 事前拦截**（用户选了全免费判官 + 完整中文存档，这两个选择是自洽的）：
> - 事前拦截：**弱**——全免费 GLM 判官，证据说它抓不出失真（TNR<25%）
> - 事后可查：**强**——英文原文 + 中文逐字全译 + 每句时间戳，全部永久保留
>
> **关键洞察（第 7 轮用户提出，我之前漏了）**：溯源到**英文**原文对本用户等于没溯源——愿意啃英文就不需要这个产品了。**溯源必须溯到中文才有实际价值。**
>
> **关键洞察 2（第 11 轮用户提出，我又漏了）**：v1 精华稿「金句集锦式」——用户反馈**「看不懂整篇在讲什么，每段都很突然」**。定位修正：**精华 = 给「想了解但不熟」的读者的导读**，不是给圈内人的猛料集锦。好懂需要 AI 补背景（原话没有的内容），这跟「怕四不像」冲突——**由「AI 补充明确标注 + 四层溯源兜底」化解**：好懂 ≠ 失真。

## User Stories      ← 中海拔(可读)+ Gherkin(精确)挂底下
> 第 26 轮起写。优先级：**P0=v1 必须** / **P1=v1 之后** / **二期**。四页线框已双关通过，US 引用其 ASCII 命名元素。
> 用户 WHO=本人+身边小圈子；核心 PAIN=英文长视频看不完+整集翻译质量差；核心主张=浓缩可信+知识串联。

### ── 浏览发现流（列表页）──

### US-1 · 看最新精华  [P0] 🔒
**As** 订阅者 **I want** 一进站就看到最新处理好的单集精华卡片流 **So that** 不用找、第一时间知道有什么新内容
**实现页面**: 列表页
**验收 (Gherkin)**:
  Given 已有若干集处理完成
  When  我打开站点首页
  Then  默认按发布日期倒序显示「单集卡」流（封面/标题/嘉宾+身份/一句精华语录/来源/日期）
  And   已读的卡片整体压暗、未读正常
  Scenario 空站（一集都没有）:
    Given 系统刚上线、还没处理出任何一集
    When  我打开首页
    Then  [异常] 显示友好空状态（如"还没有内容，订阅源更新后自动出现"），不是空白页

### US-2 · 按标签筛选  [P0] 🔒 (第26轮全进v1)
**As** 订阅者 **I want** 点标签只看某一类的集 **So that** 关注某个主题时能收窄
**实现页面**: 列表页
**验收 (Gherkin)**:
  Given 列表页顶部有「标签条」（AI 自由打的标签）
  When  我点某个标签（如 #Agent）
  Then  列表只显示带该标签的单集卡，顶部显示可清除的筛选条件
  When  我点「清除」
  Then  恢复全部、最新排序
  Scenario 筛选无结果:
    When  某标签下（因其他条件叠加）无集
    Then  [异常] 显示"没有匹配的集"+一键清除筛选

### US-3 · 搜索  [P0] 🔒
**As** 订阅者 **I want** 搜标题、人物、概念 **So that** 直接找到想看的
**实现页面**: 列表页顶栏搜索框（各页可达）
**验收 (Gherkin)**:
  Given 顶栏有搜索框
  When  我输入关键词
  Then  返回标题/人物/概念命中的结果
  Scenario 搜索无结果:
    Then  [异常] 显示"没找到相关内容"，不报错

**As** 订阅者 **I want** 点开一个人，看他在所有集里说过的精彩的话 **So that** 得到"这个行业最精英人士说过什么"的知识库

  And   每条金句可点"回原集"跳回出处那一刻（溯源）

### US-9 · 看某实体出现在哪些集  [P0] 🔒
**As** 订阅者 **I want** 看一个人/公司/概念出现在哪些集、担任什么角色 **So that** 全面了解 TA

**验收 (Gherkin)**:
  Given 打开某实体页
  Then  「出现在这些集」自动列出所有相关集，可按角色（作为嘉宾/被提及）筛
  And   「关联实体」列出常一起出现的人/公司/概念，可点跳转
  And   「关系图」显示该实体邻域，点节点跳转
  And   [系统] 这些区块由显式链接+属性自动生成（反链/图谱白送，前提是流水线写了链接）
  Scenario 实体只出现在 1 集:
    Then  [异常] 正常显示单条，不因数据少而破版

### US-10 · 全局图谱漫游  [P0] 🔒 (第26轮全进v1)（价值存疑，先做后验）

### ── 贯穿主张：可信/溯源 ──

### US-11 · 精华可回原核对  [P0] 🔒  ← 核心主张兑现
**As** 对"AI 会不会瞎编"存疑的订阅者 **I want** 任何金句/论点都能回到原片核对 **So that** 我信得过这份精华、"四不像"了也查得出
**实现页面**: 详情页金句时间戳 / 实体页金句墙"回原集"
**验收 (Gherkin)**:
  Given 精华里的金句带时间戳、实体页金句带"回原集"
  When  我点时间戳/回原集
  Then  跳到该集详情页对应位置（金句原声/原文可核）
  And   [系统] 金句在生成时经机器闸门校验（引语逐字命中转写稿+时间戳包含+说话人匹配）
  #     ▸ 留痕改（2026-07-19 用户授权 · ADR 0013）：时间戳+说话人已降软提醒（检测但不拦），硬校验只剩「引语逐字命中转写稿」；无时间戳源金句标「来自原文」。原三联表述保留、此为收窄留痕。
  And   [系统] 全译+英文原稿云端存档，极端存疑时可调取（不在页面 UI）
  # 主张表述：不宣称"零失真"，宣称"精华可能有偏差，但可溯源可核对"（第 7/11 轮 🔒）
