## 二、🔴 最重要的发现：机器闸门抓到的是**我**

**首次跑闸门：7 条金句只过了 4 条。**三条被卡，性质完全一致——**我把相隔十几秒的两条发言，拼接成了一句连续引语**，中间跨过了主持人的插话。修正后再跑，又抓出第 4 条同类错误（[43:23] 那句「恐龙」）。

举例：
- 我写：「人们不看代码了……这件事现在可能比看代码本身更重要。」标 `[05:51]`
- 实际：`[05:44]` 一句 + swyx 插「Yeah」+ `[05:51]` 另一句
- 另有一条时间戳标到了**主持人提问的那一秒**（48:31 是 swyx 在问，Akshat 那句在 48:34）

**四个错误的意思全是对的，我没编造任何内容。** 但「他没那样连续说过这句话」，点时间戳回去听会对不上。

> ⭐ **这四个错误，LLM 判官几乎不可能抓出来**——拼接后的引语更流畅、意思更清楚，判官只会觉得质量更好（正是 调研-裁判可靠性.md 说的「判官偏爱流畅行文」）。而字符串匹配一秒就逮住了。
>
> **这是「机器闸门是唯一不会说谎的一层」的实证**，也说明：**引语拼接是 LLM 的系统性倾向，不是偶发手滑**（4/12 命中率）。最终 12/12 通过。

## 三、🔴 R2 风险实证：官方文字稿自己就有错

6 个翻译 agent **各自独立**撞上同一类问题：
- `Modal` → `model`（多处：`model philosophy`、`the next phase of model`）
- `foundation models` → `fund models`｜`Claude 4` → `Cloud 4`｜`Nsight profiler` → `inside profiler`
- `Data Council` → `Data Console`｜`highest order bit` → `highest order bid`｜`Modal Sandbox` → `Model Sandbox`
- 护栏金句原文写作 `LLM media permission`，实为 `LLM-mediated permission`
- `a hard infer problem`（疑为 infra 口误）｜`English and prompting is the fourth language`（前文只提两种语言，`fourth` 是口误）

> **我们连 Whisper 都没用上，官方稿就已经错了** → 风险登记 R2「机器闸门保的是转写稿不是原话」得到实证。**英文原稿层必须永久保留——它才是真相。**

## 四、❓ 新决策点：翻译该不该纠错？（待用户拍板）

**agent 们选择照错译**（`model philosophy` 原样保留不译成 Modal），**我在精华稿里顺手纠正了**（`LLM media` → 「由 LLM 裁决」）。**两边不一致，需要一条规矩。**

三个选项：
- **A 照错译 + 标注**：忠实搬运，错处加 ⚠️ 标注说明疑似原稿错误（agent 的做法 + 我在金句处的补救）
- **B 直接纠正**：译文里改对，不标注（读起来最顺，但读者不知道原稿有错）
- **C 纠正 + 标注**：译文改对，同时标注「原文写作 X，疑为转写错误」（信息最全，最啰嗦）

## 五、其他实测数据

- **闸门误报 1 处**：「代理」被判违规（术语表要求 agent 不译成「代理」），实为 `middle proxy` 的正确翻译 → **闸门会有误报，需人扫一眼**
- **术语一致性**（6 段并行翻译的最大风险）：沙箱 35 次、推理 34 次、装饰器 4 次、投机解码 3 次——**跨段完全统一**，术语表机制有效
- **数字保真**：17 云 / 十万沙箱 / 3 Tbit/s 全部正确传递
- **压缩比**：68,294 字符英文原稿 → 8 分钟精华稿（约 7:1）
