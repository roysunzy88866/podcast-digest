# 风险登记册

> 建立于第 7 轮（2026-07-15），触发：用户问「整个流程里哪些不确定性比较大」。
> 分级依据：**能不能靠规划消除**——这是最有用的分法。规划能解的现在解；只能靠跑一期知道的，别装作规划得出来。
> 每条标：证据强度 + 当前状态。持续更新。

---

## 🔴 第一档 · 已证伪（我之前的设计错了，必须改）

### R1. LLM 裁判基本是安慰剂 ⭐⭐⭐
**原设计**：DeepSeek 写稿 → GLM 免费 key 当裁判审「有没有偷换意思」。
**证伪证据**【强，见 调研-裁判可靠性.md】：
- LLM judge **TPR 96% / TNR <25%**（[arXiv 2510.11822](https://arxiv.org/abs/2510.11822)，2025-10）——认出「好东西」96 分，认出「坏东西」不到 25 分。**我们要的恰恰是抓错的能力。**
- 零样本 prompt 做忠实度检测，**GPT-4-turbo 只有 63.4% F1**（[LettuceDetect](https://arxiv.org/abs/2502.17125)，2025-02）；GLM-4.7-Flash（AA 23）档次远低于 GPT-4-turbo → 只会更低。
- 判官弱于被判者时，**分歧中 96.4% 是判官错**（[arXiv 2601.19532](https://arxiv.org/abs/2601.19532)，2026-01）。GLM-4.7-Flash（AA 23）vs DeepSeek V4-Flash（AA 40）差 17 分，正落在警告区间。
- 判官**偏爱流畅行文**，而语义偷换恰恰藏在流畅行文里。
- GLM-4.7-Flash 在**任何** faithfulness benchmark 上无公开成绩【未验证 = 无法预判】。

> 🚨 **定性**：它是「让人感觉安全」的装置，不是真闸门——**最危险的一类设计**。
**处置【🔒 第 7 轮用户拍板】**：用户选**坚持全免费（只用 GLM 判官）**，并接受主张相应降级。配套：
- 主张改为「**精华可能有偏差，但原文和全译完整保留，随时能查**」——把主张对齐到证据，而非把证据美化到主张
- 用户同时拍板新增**中文逐字全译层**（R17）→ 副作用是把此处判定从跨语言变**同语言**，GLM 中文强项对口，**部分对冲了本风险**
- 判官用法仍守证据：只做二元判定 / 点对点喂片段 / 位置随机化；**它说「没问题」不记作通过**（TNR<25%）

### R18. LLM 系统性地「拼接引语」 ⭐⭐⭐【第 8 轮试跑实证发现】
**现象**：试跑第 1 期，12 条金句里 **4 条是拼接**——把相隔十几秒的两条发言接成一句连续引语，中间跨过主持人插话；另有 1 条时间戳标到了主持人提问的那一秒。
**性质**：**意思全对、没编造内容**，但「他没那样连续说过这句话」，点时间戳回去核会对不上。
**关键**：这是**最强模型在认真工作时**仍然犯的错，且 **4/12 的命中率说明是系统性倾向而非偶发**。
> ⭐ **LLM 判官几乎不可能抓出来**——拼接后更流畅、意思更清楚，判官只会觉得质量更好（印证 调研-裁判可靠性.md：判官偏爱流畅行文 + TNR<25%）。**字符串匹配一秒逮住。**
**处置**：机器闸门必须包含「引语逐字命中 + 时间戳所在发言必须包含该句 + 说话人匹配」三项联合校验（已实证有效，修正后 12/12）。**这条是「①层是真闸门、②层是安慰剂」最硬的实证。**

### R2. 机器闸门保的是转写稿，不是原话 ⭐⭐ 【第 8 轮试跑：已实证发生】
**漏洞**：第一级闸门逻辑=「引语必须逐字存在于转写稿」。但 **Whisper 在静音/停顿处会无中生有**（[ACM FAccT 2024](https://www.healthcare-brew.com/stories/2024/11/18/openai-transcription-tool-whisper-hallucinations)，幻觉率报告 1%-80%）。
→ **如果转写稿里有一句嘉宾从没说过的话，机器闸门会「验证通过」**。整条防失真链的地基可能已经错了。
**证据强度**：**已实证**（第 8 轮试跑）——6 个翻译 agent 各自独立撞上官方文字稿的转写错误：`Modal`→`model`、`foundation`→`fund`、`Claude 4`→`Cloud 4`、`Data Council`→`Data Console`、`highest order bit`→`highest order bid`、`LLM-mediated`→`LLM media`。**我们连 Whisper 都没用上，官方稿就已经错了。**
→ 直接推论：**英文原稿层必须永久保留，它才是真相**；中文全译是派生物。
**处置**：① VAD 先切静音 ② 考虑 gpt-4o-transcribe（开发者实测幻觉基本消失【个人实测口碑】）③ **金句人工抽检回听原音频**——这是唯一能真正兜底的（见 R3 结论）。
**主张影响**：「零失真」是夸大。实事求是=**「失真可追溯、可抽查」**。

### R3. 天花板：没有「无人值守」的解 ⭐⭐⭐
**证据**【强】：忠实度检测 SOTA 排行榜（[LLM-AggreFact](https://llm-aggrefact.github.io/)）**榜首也只有 77.4%**。对所有方案一视同仁，包括加强后的方案。
**含义**：**自动层的作用是把人工抽检范围从 100% 缩到 20%，不是缩到 0%。**
**处置【🔒 第 7 轮用户拍板】**：用户**不要抽检环节**，改为**完整存档、事后可查**（英文原文 + 中文全译 + 每句时间戳全保留，「以后我会查」）。
> **这是换了防失真哲学，不是回避风险**：事前拦截弱（认了），事后可查强（全留着）。对个人自用是自洽的权衡——**且比反过来诚实**。
> **残留风险**：用户不主动查的那些期，偏差就一直在那儿不会被发现。用户已知情接受。

### R4. 免费专用检测器用不了（跨语言） ⭐
**发现**：忠实度检测的真正 SOTA 是**专用小模型**（0.1B 的 HHEM 打平 GPT-4；0.4B 的 LettuceDetect 79.22% F1 吊打 GPT-4-turbo 的 63.4%）——**但免费的全是英文单语**，我们是英文→中文跨语言，用不了。唯一支持简中的 HHEM-2.3 是收费版。
**跨语言额外风险**：[CCHall](https://aclanthology.org/2025.acl-long.1485.pdf)（ACL 2025）——跨语言幻觉是独立失败模式，常表现为错误翻译（"stand"→「站在」而非「忍受」），**这正是「偷换意思」在本项目的典型长相**。
**处置【第 7 轮部分缓解】**：中文全译层（R17）让**浓缩环节**变成同语言判定，跨语言风险被**挤压到翻译环节这一处**（而非弥漫全链）。专用检测器仍用不了（它们是英文的，我们判的是中文对中文）；但至少 GLM 的中文强项对口了。**翻译环节本身的跨语言风险，由「保留英文原文 + 用户事后查」兜底。**

---

### R17. 翻译环节在最上游，翻错了后面全错 ⭐⭐【第 7 轮新增】
**来源**：用户拍板新增「英文转写稿 → 中文逐字全译」层（「以后我会查」）。
**风险**：链条变成 英文原文 →(翻译)→ 中文全译 →(浓缩)→ 精华稿。**翻译是新的上游单点**——翻错了，浓缩再忠实也是忠实于错误。跨语言幻觉常表现为错误翻译（[CCHall, ACL 2025](https://aclanthology.org/2025.acl-long.1485.pdf)：如 "stand" 译成「站在」而非「忍受」）。
**缓解**：① **英文原文永久保留在旁**，它才是真相，中文全译是派生物 ② 用户以英文原文事后兜底（用户明确接受）③ 翻译引擎可考虑宝玉三步翻译法（直译→反思→意译，质量最好但贵 3 倍，见 调研-同类产品摸底.md §7）。
**架构收益（对冲）**：此层让**浓缩环节的判定从跨语言变同语言**（中文全译 → 中文精华），GLM 判官的中文强项终于对口，机器闸门的引语匹配也变成同语言匹配（更硬）。**净效果大概率是正的。**

## 🟡 第二档 · 纯赌（规划再多也没用，只能跑一期才知道）

### R5. 浓缩质量本身 ⭐⭐⭐ ← 最大的不确定性
**性质**：整个项目的价值主张就是「浓缩不失真」，**但我们从没跑过一期**。三层金字塔、时间戳绑定、机器闸门——全是纸上的。
**前车之鉴**：Opus Clip 用户实测**约 40% 生成片段直接扔掉**（[BIGVU 实测，2026](https://bigvu.tv/blog/opus-clip-tested-2026-where-ai-wins-40-percent-discard/)）——「AI 自动选段」本身就不可靠。
**处置**：**这条决定项目生死，且无法靠继续规划消除**。建议尽早跑通一期真实内容看效果，再决定要不要建全套系统。

### R6. TTS 听感 ⭐
Azure 中文音色「一线水准」是我的【口碑印象】，本次 TTS 横评未含 Azure。**验收=你听第一期**，不满意换 MiniMax（¥1-2/期）。

### R7. Mac mini 本地转写速度 ⭐
型号/内存未知（open-question #11）。2 小时音频转写+说话人分离要多久没实测。**不影响方案成立，影响体验**（是否要等一晚上）。

### R8. 选题过滤的品味 ⭐⭐
AI 能不能替你判断「这期值不值得看」？**冷启动没有你的反馈数据**。AI HOT 是 5 维评分 + 人工反馈机制训练出来的，我们没有那个数据。
**处置**：第一版可先不过滤或粗过滤（只按关键词），攒够反馈再谈智能过滤。

## 🟠 第三档 · 我之前说得太乐观，需修正

### R9. 播客 RSS「全白色通道」的说法过于轻松 ⭐⭐
**我之前说**：播客 RSS 音频直链下载「是播客 App 的标准工作方式，完全白色」。
**修正**：**下载**来收听是标准的；但**下载→翻译→浓缩→再分发到自己的 feed**，性质不同——这是二次加工与再分发，不是个人收听。
**实际风险**：私有小圈子（不可猜测的 feed URL、不公开索引）风险极低；**但不是「完全白色」**，属灰色自担。
**处置**：已知情即可（用户第 2 轮已接受 ToS 灰色自担）；**但不要公开分发、不要做成公开产品**——那是另一个风险量级。

## ⚫ 第五档 · 外部变化（不可控，需设计容错）

### R14. 模型下线是常态 ⭐⭐
**实证**：
- 🚨 `deepseek-chat` / `deepseek-reasoner` **2026-07-24 15:59 UTC 停用**（[官方](https://api-docs.deepseek.com/quick_start/pricing/)）——就是下周
- GLM-4.5-Flash **已于 2026-01-30 下线**，自动路由到 GLM-4.7-Flash（[官方](https://www.zhipuai.cn/zh/news/148)）
**含义**：设计文档里写死的模型名，几个月就会烂。**将来实现时模型名要可配置，不要硬编码。**

### R15. YouTube 反爬是移动的靶子 ⭐
PO token 已蔓延到字幕接口且按视频签发；管道必须内置「yt-dlp 自动更新 + 失败告警 + 人工补抓」。
**缓解**：播客 RSS 成为主源后，此风险已从「致命」降为「兜底源偶尔断」。

### R16. 免费额度的隐性成本 ⭐
GLM 免费档 **1 QPS**。若要靠多次投票稳住判决（需 11-15 次重复），一篇稿排队 **5.5 分钟起**。「免费」变成时间成本。

## 📊 一句话总结

**真正的不确定性只有一个半**：
- **一个** = R5 浓缩质量（决定生死，只能跑一期才知道）
- **半个** = R1+R2+R3 防失真的实际强度（证据说：自动层天花板 77.4%，人工抽检不可省）

**其余都是工程问题或已验证。** 结论：**继续规划的边际收益已经很低，该跑一期真的了。**
