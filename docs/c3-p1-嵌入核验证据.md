## 一手证据(真 build 出来的 HTML)

**输入** `p1-embed.md`:
```markdown
![[p1-source#^p1testquote]]
```
**输出**(`public/p1-embed.html`,原样摘录):
```html
<blockquote class="transclude" data-url="p1-source" data-block="#^p1testquote" data-embed-alias>
  <p id="p1testquote">「为什么你要让一个智能体去阅读数百个 Kubernetes 文件」
— Akshat Bubna [05:06]</p>
  <a href="./p1-source#p1testquote" class="internal internal-link transclude-src">Link to original</a>
</blockquote>
```

**读出三件事**:
1. **只拉了那一个块**。源页里紧邻的「这是一段无关的正文,不该被嵌入。」**没有**被拖进来 → 嵌入精确,金句墙不会把邻近段落带出来。
2. **`<a ... class="transclude-src">` 是 Quartz 自动加的** → **US-8 验收原文「每条金句可点『回原集』跳回出处那一刻」不用自己实现**,块嵌入白送。锚点 `#p1testquote` 直落金句那一段。
3. **`^p1testquote` 只出现在 HTML 属性里**(`data-block`),不是给读者看的文字 → 无语法残留。

> ⚠️ **首跑差点误判**:我第一版 fixture 里同时放了 `![[p1-source]]`(**整页**嵌入)做对照,导致「无关正文被拖进来」「残留 `^块ID` 字样」两个假阳性——前者是整页嵌入的正常行为、后者是 HTML 属性被我的粗正则扫到。**按段隔离重读 HTML 才拿到真结论**。记在这里:验收信号本身也会骗人,粗正则 grep 不算证据。

## 顺带验出的两个待办(不在 P1 结论内,C3 实现时处理)

1. **块链接的显示文字很丑**。`[[p1-source#^p1testquote]]` 渲染成链接文字 `p1-source > ^p1testquote`(实测原文)。→ 金句墙若用**链接**而非嵌入,必须走别名管道 `[[集#^块|回原集]]`。**本片用嵌入,主路径不受影响**。
2. **站点 locale 是 `en-US`,但这是个中文站**(`site/quartz.config.yaml`)。所以上面那条溯源链接现在显示英文 **"Link to original"**。Quartz 自带 `zh-CN` locale,该串译作**「指向原始笔记的链接」**——比英文强,但对播客产品仍别扭(是「集」不是「笔记」),而 US-8 要的是**「回原集」**。→ C3 Scenario 4 做金句墙时一并定;**Quartz 定制要收进独立组件,不散改 fork**(项目 CLAUDE.md 硬规矩)。此项涉及对外文案,届时报用户。
