# ADR 0010 · C7a 去 Cloudflare Access 登录门,站点完全公开

- **状态**:已定(2026-07-19 用户明文拍板)
- **相关**:drift #17 / #15 · 需求共识 CONSTRAINTS⑥(L31)· 输出层(L194)· WHO(L23)

## 背景
C7a(上云·能看能听)开工时,共识 🔒 写「音频公开输出→私密性靠 Cloudflare Access 登录门,不靠 URL 不可猜」(L194 / CONSTRAINTS⑥)。据此 C7a 原计划上 Access 加密登录门,并因此撞出一个矛盾:**站点私密(交互式登录门)vs 播客 App 按 URL 订阅**——Apple Podcasts 等是后台爬虫,过不了浏览器式登录门。

## 决策
**去掉 Cloudflare Access 登录门,站点完全公开访问(可被搜索引擎收录)。**

## 理由
1. **登录门是过度设计 + 误读 WHO(用户 2026-07-19 指出)**:WHO「小圈子私有站」是**受众定位**(自用 + 朋友、不做账号/社交/运营),**不是访问加密**。把它实现成 Access 鉴权,是把受众定位错当访问控制,且与 FORM 🔒「发链接即可分享」相冲突(Access 要逐个给朋友开授权)。
2. **假矛盾消失**:站点公开后,feed + 音频天然可达,播客 App 直接订阅,**D33 直接闭合**,「私密 vs 可订阅」不复存在。
3. **更简**:C7a 少一整块(Access 应用 + service token/bypass 等),部署链路更短。

## 后果 / 自担
- **版权敞口扩大**:内容是他人受版权播客的中文翻译 + 再合成音频(演绎)。公开 + 可被搜索引擎收录 = 从「私有小圈子分发」扩大到「公开二次分发」,可被原作者/平台发现、可能收 DMCA / 下架。**用户 2026-07-19 明文接受此风险自担**(原话:「我知道并接受公开分发的版权风险自担,按完全公开改共识、继续 C7a。」)。
- **WHO/AUDIENCE 定位不变**(仍自用 + 小圈子受众);变的只是站点技术上公开、不加密。
- **结项红线更新**:原「Cloudflare Access 私密性」项作废,换为「版权公开自担(用户明文)」持续提示,直到闭合或用户再次明文。
