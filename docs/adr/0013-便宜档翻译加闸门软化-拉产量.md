# ADR 0013 · 便宜档翻译(GLM-4.6)+ 防失真闸门软化,以拉起产量

- **状态**:已定(2026-07-19 用户明文授权,`[standard-change]`)
- **相关**:CLAUDE.md 铁律「写稿/翻译只用 GLM-5.2」「防失真三联」「机器闸门不许砍」· drift-log #25 · 需求共识「防失真三联」留痕
- **背景**:项目上线两天仅产出 1 集内容,用户强烈要求拉起产量。逐层排查出三个堵点,均经用户明文授权后改。

## 三处标准变更(逐条)

### ① 翻译改用 GLM-4.6(原铁律锁 GLM-5.2)
- **原铁律**:写稿/翻译只用 GLM-5.2,免费档只当判官(依据:GLM-4.7-Flash 实测编造原话,试跑记录-第2轮)。
- **改**:`translate.mjs` 默认模型 GLM-5.2 → **GLM-4.6**;整理/浓缩(condense)仍 GLM-5.2。
- **为什么安全**(实测坐实,gray-swan 对比):condense 输入是**双语对齐稿**(英文 ‖ 中文),保真关键项全不依赖翻译模型 ——
  金句英文侧从**英文**逐字摘(闸门①核)、金句中文由 **GLM-5.2 在 condense 里重译**、专名/数字**回英文**核。翻译只当"供 GLM-5.2 理解"的中文脚手架,而 GLM-5.2 看得到英文能交叉核对。实测导读质量与 GLM-5.2 全程版持平、无编造。
- **附带好处**:翻译是 GLM-5.2 调用大头(每集 35+ 块),挪到 GLM-4.6 → GLM-5.2 只剩 condense/judge 几次 → **根治 429 限流**;成本也降。
- **淘汰 GLM-4-Flash**:实测 50 段编号批译接不住、反复漏段硬挂(同 4.7-Flash 能力不足),不用。
- **残余风险(用户知情接受)**:导读一般性叙述(非专名/数字)不过闸门;若便宜翻译把某句意思译歪、GLM-5.2 又没核出,可能漏一句偏的进导读。概率因双语对齐而低、非零。缓解:condense prompt 补「中英冲突时以英文为准」(待办)。

### ② 金句「说话人匹配」+「时间戳」从硬拦降为软提醒
- **原铁律**:金句 = 逐字命中 + 时间戳区间 + 说话人匹配,三项联合硬校验;机器闸门不许砍。
- **改**:`gate.mjs`③说话人、`gate-facts.mjs` D8 内联时间戳张冠李戴、以及金句②时间戳 —— **全降为「待核」软提醒,不再拦发布**;硬保真只剩**①逐字命中转写稿**(仍挡 GLM 凭空编造金句 —— 必须逐字连续出现)。
- **为什么**:自动说话人推断在多人访谈里配错标签 → 闸门③大量拦(自伤);且用户要接第三方无时间戳稿(SingjuPost),时间戳硬拦会全拦。用户明文:「大致判断这个人是谁就 OK」「不考虑时间戳,标注来自原文即可」。
- **残余风险(用户知情接受)**:失去「谁说的/哪一刻说的」校验,偶发张冠李戴/错时间可能上公网。

### ③ D17 专名比对降误报(不砍"挡编造"的本事)
- **改**(`gate-facts.mjs`):a) **版本化实体容错** —— 导读「AlphaFold」⊆ 转写稿「AlphaFold3」,只剥尾部数字、词长 ≥5 才放行(短词不放,防 Sam⊆Samsung);b) 跳单/双字母(G/H 数学记号);c) 通用缩写进白名单(CTO/CEO/API/GPU/LLM…)。
- **不砍本事**:编造一个真相源里根本没有的**全新名字**仍被挡。实测:axiom 的「Srinivasa/Kontorovich」转写稿 0 次命中 → GLM 真编造,闸门**正确拦下**;而 AlphaFold(误报)、CTO(缩写)清掉。
