# C5 P1 · 中文搜索核验证据(US-3 地基)

> 2026-07-18。C5 开工前置核验:Quartz 白送搜索对中文行不行(官方一面之词、未实测,踩过 Bases/图谱「装着≠好用」的坑,先验)。**结论:通过(真 build + 浏览器实测)。**

## 实测方法(不认 mock 绿)
- 真 build 站(38 文件→157 emitted)+ `build --serve`(localhost:8080)+ 浏览器真打开集页 → 点搜索 → 输中文词 → 看结果。
- 测两个不同类型的词:①常见概念「智能体」②具体 4 字技术词「投机解码」(防单字碰巧)。

## 结果:两词都精准命中、排序合理、高亮正确
| 搜索词 | 首条结果 | 一并命中 | 判定 |
|---|---|---|---|
| 智能体 | `智能体 (agent)` 实体页 | 讨论它的两集 + `智能体体验` 标签 + 相关概念(沙箱/上下文策略) | ✅ 精准,exact 排最前 |
| 投机解码 | `投机解码 (speculative decoding)` 实体页 | 右栏带出「集里怎么说它…让小模型先猜、大模型验证的加速推理」+ Modal 集 | ✅ 精准,4 字技术词非单字碰巧 |

- 右栏预览把命中词高亮(截图实测)。人物(Akshat/swyx,拉丁名)与标题词(搜「智能体」返回集标题)也在索引内。

## 为什么能用(机制)
- 本 fork 搜索来自插件 `github:quartz-community/search`(FlexSearch 0.8),**自定义 encoder**(`.quartz/plugins/search/src/components/scripts/search.inline.ts:26-68`)专门识别 CJK 区段(平假名/片假名/CJK 统一表意 0x4e00-0x9fff/谚文/CJK 扩 B),把中文**按单字切 token** 再 `tokenize:"forward"` 前缀匹配 → 中文可搜。默认 FlexSearch(空格分词)对中文无效,此插件已解决。
- **FlexSearch 本地打包**(`public/postscript.js` 157KB 内含),**非 CDN** —— 搜索这块 C7 离线/Access 部署无外部依赖。

## 顺带发现(记账,非 P1 阻断)
- **首页 `/` 当前 404**:现在没有列表页(正是 C5 US-1 要建的),集页/实体页都 200。预期内,C5 主交付补上。
- **其它前端库仍走 CDN**(build 产物实测):图谱 `pixi.js@8`、`d3@7`、数学 `katex`、图表 `mermaid` 来自 jsdelivr/cdnjs。**C7 红线相关**:Access 私密站 + 严格 CSP 下运行时外链 CDN 会泄漏使用痕迹/可能被拦 → 需评估本地打包。已记 **D36**。(搜索的 FlexSearch 不在此列,已本地打包。)

## 诚实边界
- 2 集语料上「精准」;**50 集规模的搜索质量(精度/排序/单字切噪音)要灌量后复验**(US-3 的 50 集大验收,单列批次)。本 P1 证「中文搜索机制通、能用」,不证规模质量。
