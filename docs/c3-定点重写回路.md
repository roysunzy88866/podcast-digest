# C3 · 事实层定点重写回路(`scripts/repair-facts.mjs`)

- **日期**:2026-07-17 · **状态**:已跑通并真修掉一条活的编造 · **用户 2026-07-17 拍板选此方案(选项 B)**

## 为什么要有它:整篇重摇是老虎机(实测,不是推演)

原来的修法 = 「改 prompt → 整篇重跑浓缩 → 祈祷全过」。集 2 连摇三轮的实测:

| 轮次 | 改了什么 | 闸门逮到的**新**编造 |
|---|---|---|
| 1 | — | `[06:25 / 31:16 Reynold Xin]` 两处移花接木(实际是主持人在说) |
| 2 | prompt 加 GCP/Azure/Devin/2021-2022 反例 | **Ghodsi**(转写稿只说过「Ali」,它补了姓氏) |
| 3 | prompt 加「不许补全人名」规则 | **2024**(「在 2024 年的 Data AI Summit 之后」,转写稿里 2024 与拼写年份全无) |

**每轮堵掉上一类,又冒出新的一两条**;每轮烧约 2 元;而且**把用户验收过的文本整篇换掉** —— 集 1 一次重摇就丢了「Modal 是什么」的定义、推理拐点(GPU:CPU 8:1→1:1)、全集唯一一处观点交锋,还新增了一条更难查的编造(把相隔 22 分钟的两个例子焊成一句)。

⇒ 整篇重摇**不收敛**,且**误伤**。定点重写只把「闸门指出的那一段」交给模型,其余一个字不动。
这也是 **C7 上云后无人值守时唯一能 work 的架构**——那时没人盯着重摇。

## 怎么跑

```bash
node scripts/repair-facts.mjs data/episodes/<id>
# 全过 → exit 0;定点重写解决不了的 → exit 1 + 列出交人处理的条目
```

流程:`gateFacts` 报失败 → 按段聚合 → 每段连同**闸门给的失败原因**和**该时段的原文(英文转写稿+中文全译)**喂给 GLM-5.2 → 判定收不收 → 收了才落盘 → 重跑闸门 → 直到全过或 `MAX_ROUNDS=4`。

## 三条不变量(每条都是本项目栽过的坑,判据抽在 `judgePatch` 纯函数里)

| 不变量 | 由来 |
|---|---|
| **① 先校验后写** | C2 教训:condense 先写后校验 → 坏稿冲掉好稿、非幂等(GLM 20260717-011)。补丁先临时落盘过闸门,不过立刻回滚。 |
| **② 不许靠删内容过关**(补丁 < 原段 75% → 拒收) | 本轮实测:prompt 里白纸黑字写了「换个写法,不是删掉」「绝不许因为怕被拦就少讲」,**GLM 照样删**。光靠 prompt 劝不住,得机器卡。 |
| **③ 整篇复检,不许修旧带新** | 本轮实测:每轮重摇都冒新编造 → 局部修同理会。补丁修好旧的却带出新的 → 回滚。 |

**为什么判据要抽成纯函数**:埋在 GLM 调用里的逻辑做不了变异验证 —— 那正是 C2「防假绿单测自己就是假绿」的翻版。现在 `judgePatch` 有 11 条测试,**5 个变异点 4 个被抓**(第 5 个是死代码,已删,见下)。

## 🔴 GLM 独立复核逮到一个我本来会交出去的真 bug(20260717-018,判 **save**)

**倒序改段**。`paras` 的 `start/end` 偏移是在**本轮开头**按当时的 `md` 算好的。一旦某段补丁改了长度,**排在它后面的段的偏移全部失效** → 用旧偏移 splice 会把正文切烂。实测复现:

```
原文    : "第一段有问题 2024。\n\n第二段也有问题 9999。\n\n第三段没事。"
先修段0 → 再用旧偏移修段1
结果    : "第一段修好。\n\n第二段也有问第二段修好。事。"   ← 正文被切烂
```

**集 2 只有一段有问题,所以这个 bug 没咬到我** —— 正是「跑通了就以为对了」的坑:我拿一个单段样本证明了回路能用,却没证明它在多段时不坏事。修法=**倒序改**(改后面的段不影响前面段的偏移),并补测试钉死「正序会切烂」。

另两条也真:
- **`judgePatch({patch: null})` → TypeError 炸穿**。讽刺的是这条 guard 我**上一轮刚按变异验证的结论当死代码删掉**。
  ⇒ **教训:「变异验证说它是死代码」只等于「对我测试用的那些输入是死的」,不等于对所有输入都死。** 已加回 `typeof` 检查。
- **等长垃圾骗过 75% 长度卡点**:`'A'.repeat(n)` 实测被收下 —— 而闸门对「没有专名也没有数字的乱码」无话可说 → 正文被静默换成垃圾还报全过。真实触发路径不是模型发癫,是 **glm-ask 输出被 maxTokens 截断**(半句话但长度够)。已加「结尾须完整句 + 须含中文」两道。

自测中还撞到我自己一个 bug:**早退路径返回结构不全** → 「本来就全过」被报成 `exit 1`。已修,幂等性实测:重跑 → 3011→3011 字、一字未动、exit 0。

## 死代码清理(变异验证的副产品)

变异验证抓出一条**假绿**:「补丁为空 → 拒收」这条判定删掉后测试仍全绿 —— 因为**空补丁必然也是缩水补丁**,被不变量②先拦下。按「简到极致」删掉该分支,不留永远走不到、却让人以为多了一层保护的代码。(同一轮里 `gate-facts.mjs` 也删了两处同类死代码:`start > maxEnd` 被新加的 `end` 校验完全覆盖。)

## 诚实边界

- **它不会让稿子变对,只会让稿子变得「有出处」**。受 D15 制约:中文侧无硬拦,闸门保「有出处」保不了「说得对」。
- **它只修闸门抓得到的**。闸门抓不到的(D27 同形词 / 中文量级词 / 字典外中文名、D28 单点时间戳弱判据、D26【背景】免检区、D19 会计智能体那类「真坐标+假内容」)—— 它一条也修不了。
- **定位不到段落就交人**,不硬猜着乱改(`locateFailure` 返回 -1 时跳过并报出)。
- **一轮一处都没修成就停**,不无限重试烧钱(`MAX_ROUNDS=4`)。
- 修不掉的**保留已修的部分**并 exit 1 列出交人 —— 不假装全过。
